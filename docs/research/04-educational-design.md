# Educational Design Research

Source: Learning science, platform analysis, AI tutoring studies

## Core Principles (Ranked by Evidence Strength)

### 1. Do-Before-Explain
Present exercise before explanation. Slight struggle of attempting without full context primes the brain. Counter-intuitive but research-backed.
- Present scenario: "You need to accomplish X. Try it now."
- Learner attempts (likely gets it partially right)
- AI delivers explanation in context of what just happened
- Learner tries again with new knowledge

### 2. Retrieval Over Re-Exposure
Every practice moment is more valuable than another explanation pass. The act of retrieval at learning time IS practice for retrieval at application time. Reading about a command is NOT practicing using it.

### 3. Fading Scaffolding (ZPD)
- Module 1: AI demonstrates, holds hand step by step
- Module 3: AI gives goal and constraints; learner determines approach
- Module 5+: AI reviews independent attempt, provides targeted correction
Structure needs to be in exercise architecture, not just prompt tone.

### 4. Andragogy (Adult Learning)
| Principle | Implication |
|-----------|-------------|
| Need to know | Open every module with what problem this solves |
| Self-concept | Let learners skip what they can demonstrate |
| Experience | Connect to things learner already knows |
| Readiness | Realistic scenarios, not toy examples |
| Problem-orientation | "Accomplish X" not "learn about X" |
| Internal motivation | Capability growth, not point accumulation |

Rule: Never make a professional feel like a student. Frame as senior colleague, not teacher.

### 5. Bloom's Taxonomy Ladder
| Level | Exercise Type |
|-------|-------------|
| Remember | "What flag does X need to do Y?" |
| Understand | "Explain why output X happened" |
| Apply | "Use the command to accomplish this task" |
| Analyze | "This broke. Find the cause" |
| Evaluate | "Given two approaches, which fits and why?" |
| Create | "Build a solution to this novel problem" |

### 6. Kolb's Experiential Cycle (Per Module)
1. Concrete Experience — do something
2. Reflective Observation — notice what happened
3. Abstract Conceptualization — connect to underlying model
4. Active Experimentation — apply in new variation

## Progression Patterns

### Competency-Based (Not Time-Based)
- Diagnostic entry point at course start
- Let advanced learners skip by demonstrating competency
- Track competencies, not completion percentages

### Progressive Skill Disclosure
- Wave 1: 20% of tool that handles 80% of tasks
- Wave 2: Variations, flags, edge cases
- Wave 3: Power features, composition
- Wave N: Rare but important, unlocked when Wave N-1 exhausted

### Spaced Repetition in Structure
- Interleaved topic cycling (A1 → B1 → C1 → A2 → B2)
- Cumulative review gates (each checkpoint includes prior modules)
- Pre-task retrieval cues ("Before we start, recall what we covered about X")
- Increasing intervals (end of module, start of next, two modules later)

## Exercise Archetypes

### 1. Guided Execution (Early)
AI gives exact steps. Learner executes, reports what they observe. Builds mechanical familiarity.

### 2. Constrained Problem (Mid)
AI gives goal + constraints. Learner figures out which features to use. Bridges knowing-about to knowing-how.

### 3. Broken State (Mid-Late)
Broken/misconfigured scenario. Learner diagnoses and fixes. Builds debugging/recovery skills.

### 4. Open-Ended Build (Late)
Realistic goal without specifying approach. Integration and transfer. Closes every major unit.

Ratio shift: heavy 1→2 early, heavy 3→4 late.

## Assessment Approaches

### Socratic Method
- "What would happen if..." — probes underlying model
- "Walk me through your approach before executing" — reveals gaps
- "Why did that work?" — distinguishes understanding from luck
- "How would you modify for edge case X?" — probes transferability

### Formative Feedback (Never Grade)
- Specific, targeted: "That worked, but there's a more idiomatic approach..."
- Comparative: "Here's how an expert would do this. What would you change?"
- Process-focused: "You tried three approaches. What led you to change direction?"
- Partial correctness: Avoid binary right/wrong

### Anti-Tedium Rules
- Ratio: 1 min explanation → 3-5 min doing
- Novelty at each exercise (same concept, different scenario)
- Explicit progress narration ("You can now do X that you couldn't an hour ago")
- Authentic stakes (real-world-shaped scenarios)
- Variable challenge (alternate easier and harder)
- Natural stopping points (30-min sessions feel complete)

## Narrative Throughline
Single "capstone project" defined at start. Each module adds one layer. By end, learner has built something real. Creates context and transfer.
